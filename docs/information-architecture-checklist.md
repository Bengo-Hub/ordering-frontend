# Information Architecture Review Checklist

Use this checklist to confirm that wireframes and IA artifacts cover every page group defined in the delivery plan.

## Public Website

- [ ] Landing page with primary call-to-actions, featured restaurants, and quick links
- [ ] About page (mission, partners, sustainability, testimonials)
- [ ] Services page (ordering, catering, fleet, treasury integrations)
- [ ] Contact page (support options, addresses, embedded map, lead form)

## Customer Ordering Experience

- [ ] Menu listing and category navigation
- [ ] Menu item detail with modifiers
- [ ] Cart and checkout (support prepay + COD flows)
- [ ] Order confirmation + receipt
- [ ] Order tracking timeline + map view
- [ ] Account/Profile (addresses, saved payment, loyalty, receipts)

## Driver Experience

- [ ] Shift clock-in/out and availability screen
- [ ] Order queue / delivery task list
- [ ] Navigation hand-off & proof-of-delivery capture
- [ ] Earnings dashboard with payouts + history

## Admin & Staff Portal

- [ ] Operations dashboard (orders, SLA, alerts)
- [ ] Order management detail view
- [ ] Rider/staff management (scheduling, assignments)
- [ ] Inventory and menu management workflows
- [ ] Promotions & campaign builder integrated with notifications service
- [ ] Treasury views (wallet balances, payouts, settlement status)

## Cross-Platform Considerations

- [ ] Breakpoints reviewed for mobile, tablet, desktop
- [ ] PWA install flows and offline states captured
- [ ] Accessibility and input modality notes (touch/keyboard)

Document gaps in the project tracking system and sync with Design if any items remain unchecked.
